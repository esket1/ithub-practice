# Гибридная система управления библиотекой

### Устройство системы
Система состоит из двух различных модулей в формате микро-сервисов

#### 1. Модуль физических ресурсов (Legacy Service)
* Стек: PHP + PostgreSQL
* Порт: 8001
* Назначение: Управление книгами, физически расположенными в библиотеке (учет книг, регистрация выдачи)
* Особенности: * Предоставляет данные по протоколу SOAP (`library.wsdl`)
    * Генерирует отчеты о просрочках через серверное преобразование XML + XSLT

#### 2. Модуль электронных ресурсов (Modern Service)
* Стек: Node.js (Express) + MongoDB
* Порт: 3001
* Назначение: Хранение каталога цифровых книг (PDF, EPUB, MP3) и их метаданных
* Особенности: * Выступает в роли API Gateway шлюза
    * Принимает JSON-запросы от фронтенда и перенаправляет их либо в MongoDB, либо в PHP-сервис по SOAP

Примечание: выбраны порты 8001 и 3001 в силу того, что на хостинге, на котором данный проект будет развернут данные порты уже заняты.
