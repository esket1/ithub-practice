## Архитектура системы

Проект реализован на базе микросервисной архитектуры с использованием API Gateway

### 1. Модуль физических ресурсов (Legacy Service)
* **Стек:** `PHP` + `PostgreSQL`
* **Взаимодействие:** SOAP (WSDL)
* **Назначение:** Учет физических книг
* **Особенности:** * Генерация отчетов о просрочках через преобразование **XML + XSLT**.
    * Предоставляет данные по протоколу SOAP (`library.wsdl`).

### 2. Модуль электронных ресурсов (Modern Service)
* **Стек:** `Node.js (Express)` + `MongoDB`
* **Назначение:** Хранение цифровых книг и файлов (PDF, EPUB, MP3).
* **Особенности:** Выступает как **API Gateway**. Принимает JSON-запросы от фронтенда и маршрутизирует их:
    * Запросы к цифровому каталогу -> к MongoDB.
    * Запросы к физическому фонду -> трансляция в SOAP-запросы к PHP-сервису.

## 🌐 Схема маршрутизации (Nginx)

Проект развернут на удаленном сервере. Настройка Nginx обеспечивает проксирование запросов в зависимости от маршрута запроса:

| Маршрут | Проксирование | Целевой модуль |
| :--- | :--- | :--- |
| `/ithub/php/` | `localhost:8001` | PHP (Legacy Service) |
| `/ithub/` | `localhost:3001` | Node.js (API Gateway) |
| `*` | `localhost:3000` | Сторонний сервер |
